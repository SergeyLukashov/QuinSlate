# QuinSlate 0.9.7
> _Last updated: 2026-07-13_

Released from `v0.9.6..72c9a1a` (5 commits). Version read from
`QuinSlate.Ui/Package.appxmanifest` (`Identity Version="0.9.7.0"`); the bump is in the
working tree, not yet committed.

## User-facing

- **Long buffers render to the last line, and the cap is now a million characters**
  (`344638c`). Side effect of the CM6 migration below: `RichEditBox` stopped painting
  glyphs past a fixed rendered height while still holding the characters
  (`Docs/Investigations/03-RICHEDITBOX-TALL-DOCUMENT-RENDER-CEILING.md`), so the tail of
  a long tab went invisible. CM6 virtualizes, so `AppConstants.MaxBufferLength` went from
  `50_000` to `1_000_000`; what remains is a sanity bound, not a rendering limit. Now
  enforced by a CM6 transaction filter (counted in CRLF form), which also covers paste
  and text drop.
- **Tabs drag to reorder** (`6cea22e`). Order persists to `settings.json` and is restored
  on relaunch; the `tabs` array's order now *is* the tab order (`SettingsService.GetTabs`
  no longer re-sorts by `Id`, and repairs malformed input on load). Consequence: strip
  position and buffer index diverge, so `TabView.SelectedIndex` is no longer a buffer
  index. `BufferKeyboardController` reads the index off the tab's `Tag`;
  `TabStripCalculator` and `TrayPeekRowBuilder`/`TrayPeekRow` follow the same rule, so
  tray peek lists buffers in the user's tab order. Platform notes in
  `Docs/Wiki/01-TABVIEW-DRAG-REORDER.md`.
- **Startup no longer pops in** (`72c9a1a`). The window used to visibly assemble itself
  over several frames. `EditorHost.PrewarmEnvironment` now starts the WebView2
  browser-process spawn first and overlaps it with the concurrent settings/buffer loads;
  first-launch `StartupService` registration (a slow broker call) is deferred until after
  the window is up; the window stays cloaked (`SetWindowCloak`) until the editor page
  reports a real paint, or a bounded `StartupRevealDeadlineMilliseconds` (800 ms) falls
  back to the flat cover. The reveal is sequenced through the page (Chromium throttles
  frame presentation while cloaked, so the uncloak instant can't be trusted), with a
  caret-blink hold so the caret never pops mid-blink.
- **Colour emoji everywhere they were monochrome** (`565799d` + `344638c`). Two separate
  surfaces, two separate causes: `FontFamily` is now pinned to `Segoe UI Emoji` on the
  tab-header emoji `TextBlock` (`TabHeaderBuilder`) and the picker button
  (`TabEditFlyoutView.xaml`), where DirectWrite's fallback was picking the monochrome
  Segoe UI Symbol glyph for U+1F170..U+1F19A; in the buffer body, the CM6 migration
  removed the `RichEditBox` forcing of the monochrome glyph.
- **Caret and Backspace step over an emoji as one character** (`344638c`), because CM6
  works in grapheme clusters. Skin-tone modifiers and ZWJ sequences no longer take
  several presses to cross or delete.

## Internal

- **Editor rebuilt on CodeMirror 6 in a WebView2** (`344638c`). Five `RichEditBox`
  instances replaced by one `WebView2` holding five CM6 `EditorState`s, served over the
  `https://quinslate.editor/` virtual-host mapping and driven by the `EditorHost` JSON
  bridge. Scoped like-for-like; the four visible differences above are the only permitted
  ones (spec 17, "Accepted improvements"), the calc highlight fade being the fourth (now
  a true alpha fade, identical end state). See ADR
  `Docs/Decisions/04-EDITOR-CODEMIRROR-WEBVIEW2.md` and
  `Docs/Specs/17-EDITOR-CODEMIRROR-MIGRATION.md`.
  - Deleted: `CalcResultAnimator`, `EditorFocusController`, `EditorPaste`, `EditorView`,
    `EditorViewBuilder`, `SmoothScrollController` (Chromium's native smooth scrolling
    replaces it). New: `EditorHost`, `EditorContentEventArgs`, `EditorContextMenuEventArgs`,
    `DitheredBackground`.
  - The vendored CM6 bundle lives under `QuinSlate.Ui/WebEditor/`; `editor.bundle.js` is
    committed, so `dotnet build` never runs npm. The "never log buffer contents" rule
    extends across the bridge.
- **Tray-peek warm-up deferred** (`72c9a1a`) behind `PeekWarmUpDelayMilliseconds`
  (2000 ms). Its first XAML-island composition steals Win32 focus for a frame even from a
  disabled, non-activating window (the island's inner input window isn't covered by the
  owner's `WS_DISABLED`), blanking the caret; two seconds in, that reads as a blink.
- **Test coverage** added in `SettingsServiceTests` and `TabStripCalculatorTests` for the
  tab-order repair and reorder rules.
- `ccee251` `README.md` Store badge repointed from the deprecated `get.microsoft.com`
  redirect to the `apps.microsoft.com` detail page. No app code.

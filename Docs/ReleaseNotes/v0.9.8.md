# QuinSlate 0.9.8
> _Last updated: 2026-07-15_

Released from `v0.9.7..v0.9.8` (6 commits).

## User-facing

- **Tray icon keeps its position across Store updates** (`e1194c5`). Windows 11 keys
  the icon's overflow/taskbar promotion by identity, which without a GUID is derived
  from the exe path; since MSIX updates install into a new versioned `WindowsApps`
  folder each time, the shell demoted the icon to the overflow on every update. Fixed
  with a fixed `TrayIconGuid` registered via `NIF_GUID` on all `Shell_NotifyIcon`
  calls. Both `Shell_NotifyIconGetRect` call sites (`TrayIcon` hover detection,
  `TrayPeekWindow` positioning) now resolve the icon by the same GUID. See
  `Docs/Investigations/06-TRAY-ICON-POSITION-RESET-ON-UPDATE.md`.
- **Emoji panel (Win+. / Win+;) works again** (`47e139c`). Edge/WebView2 silently
  dropped TSF emoji-panel commits into a `contenteditable` combining
  `autocorrect="off"` with block-level children, which CM6's `.cm-line` divs always
  are. Fixed by overriding CM6's `autocorrect` to `"on"` (spellcheck stays `"false"`).
  See `Docs/Investigations/05-EMOJI-PANEL-AUTOCORRECT-OFF-DROP.md`.
- **No more startup drop-shadow pulse** (`8a04eea`). The tray-peek window's first
  XAML-island composition during warm-up briefly stole foreground, causing DWM to
  repaint the main window's drop shadow to its inactive form and back a few seconds
  into launch. `MainWindow` now suppresses `WM_NCACTIVATE`'s inactive frame while a
  warm-up is in flight, driven by a new `WarmUpCompleted` event from `TrayPeekWindow`.
- **Peek panel background no longer bands on first show** (`6db4d46`). The peek
  window used to warm up at the default ~1139x659 DIP size, rasterising the dithered
  background at roughly triple the real 340x186 pixel count, then stretching it down
  when the real size applied, which re-quantised the dither. Now resized to real
  dimensions before warm-up starts.
- **A full buffer now tells you it's full** (`bca3c6a`). Editing a full buffer used
  to fail silently, typing simply stopped landing and an oversized paste truncated
  with no explanation. A CodeMirror transaction filter (`capFilter`) now reports
  every clamped edit over the bridge, and the host shows a throttled,
  non-interactive "Slate is full" pill (`LimitNotice` / `LimitNoticeView`) that
  resets on tab switch and panel hide. See `Docs/Specs/18-CHARACTER-LIMIT-NOTICE.md`.

## Internal

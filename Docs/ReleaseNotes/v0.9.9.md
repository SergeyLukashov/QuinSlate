# QuinSlate 0.9.9
> _Last updated: 2026-07-16_

Released from `v0.9.8..v0.9.9` (8 commits).

The editor's formatting release: tasks, lists, and links land together, and Tab
changes hands to serve them.

## User-facing

- **Checkable tasks** (`e18eb4b`). A line starting with the GFM marker `- [ ] ` /
  `- [x] ` renders as a clickable checkbox, toggled by click or `Ctrl+Enter`, with
  Notion-style `[] ` shorthand entry and Enter continuing the list. The marker text
  is the whole state: it is what lands in the buffer `.txt`, so no host or bridge
  change was needed. See `Docs/Specs/19-CHECKABLE-TASKS.md`.
- **Bullet and numbered lists** (`30fa6b7`). Typing `- ` or `1. ` at the start of a
  line converts it to an item rendered with a widget marker (a drawn dot, no font
  glyph, and right-aligned numbers). A `listRenumber` transaction filter keeps every
  contiguous numbered run sequential from 1, appending the fix to the same
  transaction so it stays one undo step. Host-origin transactions are exempt, so
  loaded text keeps its stored digits until the user edits it. See
  `Docs/Specs/20-LISTS.md`.
- **Clickable links** (`83c1c77`). `http`/`https`/`mailto` URLs render in the accent
  colour and open via `Windows.System.Launcher`. Unlike tasks and lists this writes
  nothing to the document: the link is a mark decoration over the URL's own
  characters, so the `.txt` is byte-for-byte what it would be without the feature.
  Detection requires an explicit scheme, which is what keeps a scratchpad's
  `node.js` and `README.md` from turning blue. See `Docs/Specs/21-LINKS.md`.
  - `Services/LinkService.TryCreateLaunchUri` re-validates the href host-side
    before the shell sees it, because it arrives over the bridge as buffer text.
  - `Services/AccentTextColorResolver` derives a separate `--link` colour rather
    than reusing `--accent`: the raw accent measures 3.25:1 on the dark mesh, under
    the 4.5:1 AA floor. It takes the OS shade (`AccentLight3` / `AccentDark2`, the
    same ones behind `AccentTextFillColorPrimaryBrush`), then contrast-checks it
    against `DitheredGradientBrushFactory.MidColor` and blends only if short.
- **Tab nests items; buffer cycling moves to `Ctrl+Tab`** (`f327efd`). Tab and
  Shift+Tab now indent any line and nest tasks, bullets, and numbered items under
  Notion's two rules (nest only one level under the item directly above; children
  move with their parent), with depth stored as leading spaces, two per level,
  capped at `MAX_DEPTH` 8. Plain Tab previously cycled buffers, so that binding was
  removed from `panelShortcuts.js` and cycling now lives on `Ctrl+Tab` only, which
  the tab redesign had already added. This is a behaviour change for anyone with
  Tab-to-cycle in their fingers. Nesting is shared by all three item kinds through
  `listItems.js` (recognition) and `indent.js` (the commands). See
  `Docs/Specs/06-KEYBOARD-NAV.md` and `19-CHECKABLE-TASKS.md § Nesting`.
- **Store updates no longer stall for 30 seconds** (`7ad29a6`). The Restart Manager,
  which drives Store/MSIX updates, only asks the app to close and never terminates
  it. `MainWindow`'s `WM_ENDSESSION` handler tore down without exiting, so the
  process was force-killed after 30 seconds and filed as an Application Hang. It now
  calls `Environment.Exit(App.ExitCodeNormal)` after teardown. Windows' WM_CLOSE
  fallback cannot cover this, since close is bound to hide-to-tray.
- **The About storage path fits its card at high DPI** (`6945d4b`). `AboutView`
  ellipsized the path against a hardcoded `PathAvailableWidth` of 406 DIPs derived
  from the 480 DIP window, but that is the *outer* rect: at 250% the client is 467.2
  DIPs, so the fitted string ran nearly two characters past the card and clipped.
  Width is now measured from `RootGrid` on every `SizeChanged`, with the insets read
  from the live elements rather than restated as constants. The path's own
  `NoWrap` TextBlock cannot be measured instead, as it reports the width of its text
  (629 DIPs for the full path), not of its slot.
- **Calc chains onto scientific-notation results** (`7e0c86f`). `CalcService`'s
  number parser stopped at the exponent marker, so a result it had itself formatted
  as `1.37E+52` could not be fed into a follow-up calculation. The parser now
  consumes a valid exponent suffix while still rejecting a bare `e` with no digits.

## Internal

- `5158e5f` Split the web editor's 900-line `main.js` into 15 single-purpose ES
  modules under `build/src/` (`hostBridge`, `editorSetup`, `buffers`, `charLimit`,
  `inlineCalc`, `tasks`, and others), leaving `main.js` as a thin entry point. Docs
  and the build-artifact hook that referenced `main.js` directly were updated.
- `f327efd` also added the web editor's Node test suite (`build/test/*.test.mjs`,
  run by `npm test` with no framework) plus an `editorHarness.mjs`, covering indent,
  lists, listItems, tasks, and CRLF handling.
- `7ad29a6` also refreshed `Constants/DefaultBuffers` so the shipped starter content
  shows real task checkboxes and real links, and bumped `Package.appxmanifest` to
  `0.9.9.0`. New installs only: existing users keep their buffers.

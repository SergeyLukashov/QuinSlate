# QuinSlate 1.0.0
> _Last updated: 2026-07-20_

Released from `v0.9.9..v1.0.0` (5 commits).

The 1.0 release. A user-facing theme setting, and the tray peek warm-up crash
finally closed out.

## User-facing

- **Light/dark/system theme switch** (`54e1fba`). A **Theme** submenu in the tray
  menu with three radio items (System default, Light, Dark), persisted in
  `settings.json` as `Theme` and defaulting to `System`. See
  `Docs/Specs/22-THEME-SWITCH.md`.
  - `Services/ThemeService` is the single source of truth. `Application.RequestedTheme`
    cannot change after startup in WinUI 3, so the preference is applied per window by
    setting `FrameworkElement.RequestedTheme` on each content root; `System` maps to
    `ElementTheme.Default` so a live Windows flip still tracks.
  - Long-lived windows register with `ThemeService` and get the new theme pushed on
    `Set` (`MainWindow` always, `AboutWindow` because the tray stays reachable behind
    the modal). The tray peek takes the same push via `TrayPeekWindow.SetRequestedTheme`.
    Each window's DWM frame border is retinted through a new `NativeMethods` entry.
  - Reused popup content lives outside the themed tree between opens. `TabEditFlyout`
    records the theme its cached view was built for and rebuilds on mismatch, which is
    what stops the Cancel button flashing the old colour on the first frame.
    `EmojiPicker` only re-pins `RequestedTheme` from the anchor, since its sprite
    glyphs are theme-independent images.
  - The tray menu needed a `TrayMenuFlyoutPresenterStyle` in `App.xaml` whose
    `Background` is a `{ThemeResource}`, so the presenter resolves against the tray
    helper window's element theme rather than the Windows theme. `TrayPeekPanel.xaml`
    text runs gained explicit `TextFillColorPrimaryBrush` foregrounds for the same
    reason.
- **Tray peek no longer crashes the app on launch** (`336e11d`). The peek window's
  startup warm-up hid the window on `Loaded`, mid first present, which fail-fasted
  inside `Microsoft.UI.Xaml.dll` with `0xc000027b` on slow or GPU-starved machines.
  Extracted into `Tray/TrayPeekWarmUp`, which now waits for a
  `CompositionTarget.Rendered` tick and a compositor commit before calling `SW_HIDE`.
  See `Docs/Investigations/04-TRAY-PEEK-HOVER-FAILFAST-CRASH.md`.

## Internal

- `9b60c4c` Web editor page logs and uncaught errors now reach the rolling Serilog
  sink over the WebView2 bridge, via `WebEditor/build/src/pageLog.js` and
  `Components/EditorPageLogForwarder`. Page levels mirror Serilog's; both sides clamp
  message length and cap repeats, and the never-log-buffer-text rule is preserved.
  Covered by `EditorPageLogForwarderTests` and `build/test/pageLog.test.mjs`. See
  `Docs/Specs/16-LOGGING.md`.
- `60e68af` MIT `LICENSE` added.
- `336e11d` also added the README banner asset. No app code.
- `97a6ed0` Bumped `Package.appxmanifest` and `QuinSlate.Ui.csproj` to `1.0.0`.
